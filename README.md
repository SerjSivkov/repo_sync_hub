# Repo Sync Hub

macOS-инструмент на Flutter для массового обслуживания локальных git-проектов:

- сканирование **нескольких** директорий с репозиториями;
- **Pull** ветки `main`/`master` (fetch, checkout, `git pull --ff-only`);
- **Push** в любую git-систему-приёмник (GitLab, Gitea, Bitbucket, самописный сервер);
- **Sync** — pull + push для выбранных проектов;
- обзор состояния всех репозиториев: обновления на remote, число коммитов, размер, дата последнего pull, «заброшенность».

## Требования

- macOS
- Flutter 3.12+
- `git` в PATH

## Запуск

```bash
cd repo_sync_hub
flutter pub get
flutter run -d macos
```

## Первый запуск

1. Откройте **Настройки** (шестерёнка) → **Директории сканирования** → **Добавить** —
   можно указать несколько корневых папок.
2. При необходимости настройте **систему-приёмник** (для push):
   - **Хост** — `gitlab.com`, `gitea.local`, `git.company.ru` и т.п.;
   - **Группа / namespace** — например `mobile`;
   - **Имя remote** — обычно `origin`;
   - **Токен** — опционально, если remote ещё не настроен.
3. Нажмите **Сканировать** — соберётся список всех репозиториев с `.git`.
4. Выберите проекты и нажмите **Pull**, **Push** или **Sync**.

У каждой кнопки есть подсказка (tooltip) с описанием действия.

## Отображение репозиториев

- **Группировка** — репозитории собираются по дочерней директории на 1 уровень
  ниже корня сканирования. Репозитории **с ошибками** всегда выносятся отдельной
  группой сверху.
- **Сортировка внутри группы** — по последним стянутым изменениям (свежие сверху);
  репозитории без активности **больше года** помечаются как **«заброшен»** и уходят вниз.
- **Два вида** (переключатель в шапке):
  - **построчно** — плоский список с заголовками групп; клик по шапке группы
    сворачивает/разворачивает её;
  - **деревом** — раскрывающиеся группы.
- **Столбцы-метрики** у каждого репозитория: число коммитов, размер на диске,
  дата последнего pull.
- Репозитории с доступными коммитами на remote отмечаются бейджем (число `+N`),
  после успешного pull — меткой **«получены»**.

## Операции

| Действие | Что делает |
|----------|------------|
| Сканировать | Собирает статус: ветка, default branch, dirty, remotes, коммиты, размер, дата коммита |
| Pull | `fetch --all`, checkout `main`/`master`, `pull --ff-only` |
| Push | Настраивает remote приёмника (если нужно), `git push <remote> <branch>` |
| Sync | Pull, затем Push |
| Ярлык | Симлинк на репозиторий на Рабочем столе |
| Показать в Finder / Открыть в Терминале | Открывает папку репозитория |

При незакоммиченных изменениях перед checkout можно включить **stash** в настройках.

## Кэш и расписание

- **Параллельное сканирование** — репозитории опрашиваются пулом с ограничением
  параллелизма (по умолчанию 4 одновременно, настраивается в пределах 1–16),
  что заметно ускоряет скан десятков репозиториев с `fetch`/`du`.
- **Кэш сканирования** — результаты хранятся между запусками (`SharedPreferences`).
  При старте данные показываются из кэша; свежие репозитории (в пределах настраиваемого
  TTL) при повторном скане не опрашиваются. Дату последнего скана видно в шапке.
- **Сканирование по расписанию** — в настройках можно включить периодический
  автозапуск с заданным интервалом в минутах.

## Безопасность

Токен приёмника хранится локально в `SharedPreferences`. Если remote уже настроен
в репозитории, токен не обязателен — push идёт через существующий URL.
